# LEGACY: Старт задачи с подрепо

Текущий контур работы не использует VTB-процесс. По умолчанию этот skill не применять.

Если пользователь явно запросил legacy-сценарий с VTB и подрепозиторием, действуй по шагам ниже.

**Переменные:**

- `TASK_ID` — задача в основной репе (например TASK-032), файл в `docs/tasks/TASK-XXX-*.md`
- `VTB_ID` — префикс коммитов в подрепо (например VTB-533)
- `BRANCH_NAME` — ветка в `subProjects/temp-private-2110/`

**Короткая форма от пользователя** (достаточно передать динамику):

- «Выполнить задачу TASK-032. В подрепо: ветка `feature/VTB-533/...`, задача VTB-533»
- «Начать TASK-032, подрепо: VTB-533, ветка feature/VTB-533/...»

**Действия:**

1. При необходимости создать/использовать ветку в основной репе для `TASK_ID` (см. файл задания в `docs/tasks/`).
2. В подрепо `subProjects/temp-private-2110/` работать в ветке `BRANCH_NAME`; коммиты с префиксом `VTB_ID` — см. `.cursor/rules/subProjects-temp-private-2110/commits-subrepo-vtb.mdc`.
3. Жизненный цикл задания TASK-XXX — следуй `.cursor/skills/task-management-protocol/SKILL.md` и при необходимости `reference.md`.

**Двурепозиторный сценарий (коммиты):** по умолчанию выдавай тексты сообщений для ручного коммита; коммиты от агента — только по явной просьбе пользователя (то же правило `commits-subrepo-vtb.mdc`).
