# Связка задач с GitHub Issues

## Что делать

1. При наличии issue зафиксируй его в `githubIssue` (`owner/repo#123`).
2. Добавь ссылку на `docs/tasks/*.md` в issue (или укажи путь в комментарии).
3. Поддерживай двустороннюю связь при дальнейших изменениях.
4. При смене статуса задачи синхронизируй статус issue.

## Минимальные правила

- Единый идентификатор связи: `githubIssue`.
- Если задача не связана с issue, поле можно не задавать.
- При рассинхронизации приоритет у явного решения пользователя.
