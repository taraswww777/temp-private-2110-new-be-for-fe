# LEGACY: Протокол управления заданиями (TASK-XXX)

По умолчанию этот skill не используется.

Актуальная схема:

- `.cursor/skills/task-lifecycle-lightweight/SKILL.md`
- `.cursor/skills/github-issue-linking/SKILL.md`

## Если явно запрошен legacy-режим

Полный регламент (все процедуры, шаблоны, чеклисты, примеры) — в **[reference.md](reference.md)**. Его **нужно открыть и следовать** при любой работе с жизненным циклом задания.

## Краткие обязательные пункты (не отменяют reference)

1. **Создание задания** — только по явной просьбе пользователя; файл `docs/tasks/TASK-XXX-....md` + запись в `tasks-manifest.json`, статус `backlog`.
2. **В работу** — только после явного ответа пользователя; нельзя ставить `in-progress` и создавать ветку без подтверждения.
3. **Синхронизация** — статус в файле задачи и в манифесте всегда согласованы.
4. **Завершение** — после работы не ставить `completed` без **явного** принятия пользователем (спросить критерии приёмки и «Принимаете работу?»).
5. **Коммиты** в рамках задания — префикс `TASK-XXX` (см. правила коммитов в `.cursor/rules/`).

## Где что лежит

| Что | Путь |
|-----|------|
| Манифест | `docs/tasks/tasks-manifest.json` |
| Файлы задач | `docs/tasks/TASK-XXX-краткое-название.md` |
| Полный протокол | [reference.md](reference.md) |

## Версия

Полный протокол в `reference.md` помечен версией в шапке (например 1.2.0). При изменениях протокола обновляй `reference.md` и при необходимости этот `SKILL.md`.
