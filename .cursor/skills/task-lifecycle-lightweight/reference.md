# Task Lifecycle Lightweight v1.0.0

## Цель

Единый простой процесс ведения задач без обязательного `tasks-manifest.json`.

## Формат задачи

Файл: `docs/tasks/TASK-<N>-<slug>.md`

Рекомендуемый frontmatter:

```yaml
id: TASK-6
title: Согласовать правила задач
status: backlog
source: local
githubIssue: owner/repo#123
updatedAt: 2026-04-21
```

`githubIssue` опционален. Если задача создана из issue, поле обязательно.

## Статусы

- `backlog` — задача зафиксирована, но не начата.
- `in-progress` — ведётся работа.
- `done` — завершено и принято.
- `cancelled` — отменено.

## Синхронизация с GitHub Issues

Если `githubIssue` заполнен:

1. Поддерживай взаимные ссылки между задачей и issue.
2. При переходе в `in-progress`, `done`, `cancelled` синхронизируй статус в issue.
3. При изменении статуса обновляй `updatedAt`.

## Принципы

- Не требовать обязательного исполнителя.
- Не использовать манифест как обязательный источник истины.
- Держать описание изменений кратким и проверяемым.
