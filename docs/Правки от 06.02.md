### Детализация пакета. Основная информация

Добавить поле "Дата последнего успешного копирования" -  будет заполняться, только когда копирование прошло успешно (статус пакета "Выполнено") 
При неуспешном копировании дата последнего успешного копирования не будет обновляться, будет только меняться текущий статус (статус пакета "Не выполнено")

### Детализация пакета. Работа с копированием

Нужно на основании статуса пакета, выяснить идёт ли реально процесс копирования и если он всётаки запущен, то нужно отобразить такойже прогрес бар с кнопкой отмена как и на странице ТФР.

### Список пакетов

Добавить кнопку "Отменить копирование" для каждой записи.

Отображать кнопку только если пакет находится в статусе активного копирования.

Добавить сортировку по размеру.

Если запущен процесс копирования пакетов (статус пакета "Копирование")  нельзя копировать другие пакеты - заблокировать кнопку "Копировать в ТФР".

По хорошему нужен эндпоинт по которону можно определить запущен ли процесс копирования.
или в `PackageDto` добавить атрибут `canBeCopyToTFR:boolean` - вычисляется на основе статусной модели.

При выборе пакетов и нажатии кнопки "Копировать на ТФР" проверять(на бэке), нет ли пакета с таким наименованием уже на ТФР? Если есть, то ошибка на UI

### Список заданий. Добавление задания в пакеты

На макетах отрисовать второй вариант работы.

Добавляем одно задание в несколько пакетов, в которых нет добавляемого  задания.

При добавлении задания в пакет показывать "Список пакетов" в табличном виде. (по аналогии со "Списком заданий", в форме "Удаление задания из пакетов").

Добавить сортировку по размеру (чтобы пользователь отсортировал по самому легкому пакету и добавил туда задачу)

На демо в понедельник показать оба варианта (на макетах) (текущая реализация и новый формат для сравнения).

Задание можно добавить в 1 или несколько пакетов сразу (массовый выбор)
