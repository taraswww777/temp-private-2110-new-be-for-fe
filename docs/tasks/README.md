# Управление заданиями

Эта папка содержит все задания проекта и систему отслеживания их выполнения.

## Структура

- `tasks-manifest.json` - основной манифест со списком всех заданий
- `TASK-XXX-название.md` - отдельные файлы с описанием заданий

## Формат манифеста

Файл `tasks-manifest.json` содержит массив объектов со следующей структурой:

```json
{
  "tasks": [
    {
      "id": "TASK-001",
      "title": "Краткое название задания",
      "file": "docs/tasks/TASK-001-название.md",
      "status": "completed",
      "createdDate": "2026-01-27T12:00:00Z",
      "completedDate": "2026-01-27T14:30:00Z",
      "branch": "feature/TASK-001-название"
    }
  ]
}
```

### Поля манифеста

- **id**: уникальный идентификатор задания (формат: `TASK-XXX`)
- **title**: краткое название задания
- **file**: относительный путь к файлу с описанием задания
- **status**: статус выполнения
  - `pending` - ожидает выполнения
  - `in-progress` - в процессе выполнения
  - `completed` - выполнено
  - `cancelled` - отменено
- **createdDate**: дата и время создания задания (ISO 8601)
- **completedDate**: дата и время завершения задания (ISO 8601) или `null`
- **branch**: название git-ветки для выполнения задания

## Формат задания

Каждое задание должно быть оформлено в виде Markdown документа со следующими разделами:

1. **Заголовок**: `# TASK-XXX: Название задания`
2. **Статус**: текущий статус (⏳ В работе, ✅ Выполнено, ❌ Отменено)
3. **Описание**: краткое описание задачи
4. **Цели**: список основных целей
5. **Детальное описание**: подробное описание с пояснениями
6. **Критерии приёмки**: чек-лист для проверки выполнения
7. **Ветка**: название git-ветки
8. **Порядок выполнения**: пошаговая инструкция
9. **Уточнения в процессе выполнения**: дополнения и уточнения

## Рабочий процесс

### 1. Создание нового задания

```bash
# Создать файл задания
# docs/tasks/TASK-XXX-название.md

# Добавить запись в манифест
# Отредактировать tasks-manifest.json
```

### 2. Начало работы над заданием

```bash
# Создать ветку
git checkout -b feature/TASK-XXX-название

# Обновить статус в манифесте на "in-progress"
```

### 3. Выполнение задания

- Следовать пунктам из раздела "Порядок выполнения"
- Добавлять уточнения в секцию "Уточнения в процессе выполнения"
- Отмечать выполненные пункты в "Критериях приёмки"

### 4. Завершение задания

```bash
# Создать коммит
git add .
git commit -m "TASK-XXX описание выполненных изменений"

# Обновить статус в манифесте на "completed"
# Указать completedDate
```

## Просмотр заданий

### Все задания
Смотрите файл `tasks-manifest.json`

### Выполненные задания
Отфильтруйте записи с `"status": "completed"`

### Задания в работе
Отфильтруйте записи с `"status": "in-progress"`
