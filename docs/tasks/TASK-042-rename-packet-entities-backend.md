## Детальное описание

### Контекст

Принято решение использовать название `package` для пакетов. Необходимо привести все названия сущностей на Backend к единому стандарту.

### Требования

#### Переименование сущностей

Необходимо переименовать:
- Переменные
- Функции и методы
- Типы и интерфейсы
- DTO классы
- Endpoints (если требуется)
- Комментарии и документацию

**Примеры переименования:**
- `PackageDto` → `PackageDto`
- `packageService` → `PackageService`
- `getPackageById` → `getPackageById`
- И т.д.

#### Обновление моковых данных

- Обновить генераторы моков
- Обновить структуру моковых данных
- Обеспечить согласованность с новыми названиями

#### Обновление документации

- Обновить Swagger спецификацию
- Обновить комментарии в коде
- Обновить README и другую документацию
