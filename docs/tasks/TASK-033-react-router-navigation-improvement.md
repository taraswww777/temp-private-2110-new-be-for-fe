# TASK-033: Улучшение навигации в React Router — решение проблемы с относительными путями

**Статус**: ⏳ В работе
**Ветка**: `feature/TASK-033-react-router-navigation-improvement`

## Краткое описание

Решить проблему с навигацией в React Router 5, при которой всегда нужно указывать полный путь. Текущее решение с использованием `useRouteMatch` не позволяет переходить между роутами на одном уровне или на уровень выше. Рассмотреть возможность миграции на React Router 6 или улучшить текущее решение.

---

## Контекст

В проекте используется React Router 5.3.4 (`react-router-dom`). При работе с вложенными роутами возникает проблема: для навигации всегда нужно указывать полный путь, что усложняет код и делает его менее поддерживаемым.

**Примеры текущего использования:**
```tsx
// Всегда нужно указывать полный путь
<Link to={'/report-6406/packages'} ... />
history.push('/report-6406/packages');

// Использование useRouteMatch для получения базового пути
const { url } = useRouteMatch(); // например '/report-6406'
<Link to={`${url}/packages`} ... /> // '/report-6406/packages'
```

**Проблемы текущего подхода:**
1. `useRouteMatch` позволяет получить только текущий URL, но не позволяет:
   - Перейти на роут того же уровня (`/report-6406/packages` → `/report-6406/tfr`)
   - Перейти на уровень выше (`/report-6406/packages` → `/report-6406`)
   - Использовать относительные пути (`../tfr`, `./tfr`)

2. Текущее решение требует создания методов генерации полного URL для каждого роута:
   ```tsx
   url: (basePath: string) => `${basePath}/packages`
   ```
   И подкидывать туда `basePath` из стора, что усложняет архитектуру.

**Связанный коммит:** `e93ee2506f57a375b278c71088077c5fa247e4d7` — хорошо раскрывает кейс проблемы.

## Дополнительные заметки

### Преимущества React Router 6

1. **Относительные пути:**
   - Поддержка относительных путей в `<Link>` и `navigate()`
   - Возможность указать направление навигации через `relative` prop

2. **Улучшенный API:**
   - `useNavigate` вместо `useHistory`
   - Более простой и интуитивный API
   - Лучшая типизация

3. **Улучшенная производительность:**
   - Оптимизированный рендеринг
   - Лучшая поддержка Suspense

### Примеры использования в React Router 6

```tsx
// Относительные пути
<Link to="../tfr">TFR</Link>
<Link to="./packages">Packages</Link>
navigate("../tfr");
navigate("./packages");

// Указание направления
<Link to="../tfr" relative="path">TFR</Link>
<Link to="../tfr" relative="route">TFR</Link>
```
