# TASK-060: Создание и редактирование задач с Markdown редактором

**Статус**: ✅ Выполнено
**Ветка**: `feature/TASK-060-create-edit-tasks-with-md-editor` (при необходимости)  
**Приоритет**: средний  

---

## Краткое описание

Реализовать возможность создавать задачи из интерфейса с редактированием описания в формате Markdown с использованием WYSIWYG редактора. Описание задачи должно сохраняться в файле Markdown. Также необходимо предоставить возможность редактировать описания уже существующих задач.

---

## Контекст

В текущей версии Task Viewer задачи создаются и редактируются вручную через файловую систему. Необходимо добавить полноценный интерфейс для создания и редактирования задач непосредственно из приложения с удобным редактором Markdown.

---

## Цели

- Добавить возможность создания новых задач через интерфейс приложения.
- Реализовать WYSIWYG редактор для редактирования описания задач в формате Markdown.
- Обеспечить сохранение описания задачи в файл Markdown.
- Предоставить возможность редактирования описаний существующих задач.
- Обеспечить синхронизацию между интерфейсом и файловой системой.

---

## Критерии приёмки

- [ ] В интерфейсе добавлена возможность создания новой задачи с заполнением основных полей (название, статус, приоритет и т.д.).
- [ ] Реализован WYSIWYG редактор для редактирования описания задачи в формате Markdown.
- [ ] При создании задачи автоматически создаётся файл Markdown с описанием в соответствующей директории.
- [ ] Описание задачи сохраняется в файл Markdown при сохранении задачи.
- [ ] Для существующих задач доступна возможность редактирования описания через WYSIWYG редактор.
- [ ] Изменения в описании задачи синхронизируются с файлом Markdown на диске.
- [ ] Редактор поддерживает предпросмотр Markdown в реальном времени.
- [ ] Редактор поддерживает основные возможности Markdown (заголовки, списки, ссылки, код и т.д.).

---

## Технические заметки

- WYSIWYG редактор: рекомендуется использовать библиотеку типа TipTap, React Markdown Editor, или аналогичную для React.
- Хранение файлов: файлы описаний задач должны сохраняться в `docs/tasks/` с именем формата `TASK-XXX-description.md`.
- Backend: необходимо добавить эндпоинты для создания и обновления задач, включая работу с файлами Markdown.
- Frontend: компоненты формы создания/редактирования задачи, интеграция WYSIWYG редактора.
- Валидация: проверка корректности Markdown и обязательных полей задачи.
- Синхронизация: обеспечить актуальность данных между интерфейсом и файловой системой.
