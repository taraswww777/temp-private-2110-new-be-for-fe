## Краткое описание

В форме **«Создать отчёт»** (модальное окно `CreateReportFormModal`) определить набор обязательных полей и выполнять валидацию до отправки формы. Сообщения об ошибках выводить для каждого поля (инпута) отдельно.

---

## Цели

1. **Обязательные поля** — зафиксировать перечень обязательных полей формы «Создать отчёт» (например: филиал, период отчёта, формат, тип отчёта; опционально — валюта, источник и т.д. по требованиям API/бизнеса).

2. **Валидация до отправки** — проверять значения полей до вызова API (до отправки формы). Не отправлять запрос, если валидация не пройдена.

3. **Сообщения об ошибках по полям** — для каждого инпута отображать своё сообщение об ошибке (под полем или рядом), а не одно общее уведомление. Использовать возможности react-hook-form (например, `formState.errors`, отображение `errors.<fieldName>.message` у соответствующих полей).

---

## Контекст

- Форма: `temp-private-2110/src/modules/ReportTemplateModule/components/CreateReportFormModal/`.
- Поля формы (тип `CreateReportFormModalFV`): department, reportingPeriodFrom/To, accountMask, accountSecondOrder, currency, format, type, source.
- Сейчас валидация выполняется только внутри `handleSubmit`: при отсутствии обязательных данных показывается одно уведомление «Заполните обязательные поля: филиал, период, формат, тип отчёта.» — без привязки к конкретным полям.

---

## Обязательные требования (к выполнению)

- Определён и зафиксирован набор обязательных полей формы «Создать отчёт».
- Валидация выполняется до отправки (форма не отправляется при невалидных данных).
- Сообщения об ошибках выводятся для каждого инпута (у каждого поля — своя ошибка, например под полем).
