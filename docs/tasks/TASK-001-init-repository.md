# TASK-001: Инициализация репозитория

## Статус
⏳ В работе

## Описание
Выполнить первоначальную настройку репозитория для Node.js приложения с корректной конфигурацией игнорируемых файлов для работы в различных IDE.

## Цели
1. Создать файл `.gitignore` для Node.js проекта
2. Обеспечить корректное игнорирование служебных файлов IDE (VS Code, Cursor, WebStorm)
3. Настроить игнорирование стандартных директорий Node.js проекта

## Детальное описание

### 1. Создание .gitignore
Необходимо создать файл `.gitignore` в корне проекта со следующими секциями:

#### Node.js
- `node_modules/` - зависимости проекта
- `npm-debug.log*` - логи npm
- `yarn-debug.log*` - логи yarn
- `yarn-error.log*` - ошибки yarn
- `pnpm-debug.log*` - логи pnpm
- `lerna-debug.log*` - логи lerna
- `.npm` - кеш npm
- `.pnp.*` - Yarn PnP файлы
- `*.tsbuildinfo` - информация о сборке TypeScript

#### Переменные окружения и конфигурация
- `.env` - локальные переменные окружения
- `.env.local` - локальные переменные окружения
- `.env.*.local` - переменные окружения для различных сред

#### Результаты сборки
- `dist/` - собранные файлы
- `build/` - собранные файлы
- `out/` - выходные файлы
- `*.js.map` - source maps
- `*.d.ts.map` - TypeScript declaration maps

#### IDE - VS Code
- `.vscode/` - настройки VS Code (за исключением рекомендуемых расширений)
- `!.vscode/extensions.json` - разрешить файл с рекомендуемыми расширениями
- `!.vscode/settings.json` - можно разрешить общие настройки проекта
- `*.code-workspace` - файлы рабочих областей

#### IDE - Cursor
- `.cursor/` - настройки Cursor IDE

#### IDE - WebStorm / JetBrains
- `.idea/` - настройки WebStorm/IntelliJ IDEA
- `*.iml` - модули IntelliJ
- `*.ipr` - файлы проекта IntelliJ
- `*.iws` - workspace файлы IntelliJ

#### Операционная система
- `.DS_Store` - macOS
- `Thumbs.db` - Windows
- `Desktop.ini` - Windows

#### Временные файлы
- `*.log` - логи
- `*.tmp` - временные файлы
- `*.temp` - временные файлы
- `*.swp` - swap файлы vim
- `*~` - backup файлы

#### Тестирование
- `coverage/` - отчеты о покрытии кода
- `.nyc_output/` - временные файлы nyc

#### Кеш
- `.cache/` - различные кеши
- `.parcel-cache/` - кеш Parcel
- `.next/` - кеш Next.js

## Критерии приёмки
- [ ] Создан файл `.gitignore` в корне проекта
- [ ] Файл содержит все необходимые правила для Node.js проекта
- [ ] Добавлены правила для игнорирования файлов VS Code, Cursor, WebStorm
- [ ] Добавлены правила для игнорирования системных файлов Windows и macOS
- [ ] Файл структурирован по секциям с комментариями
