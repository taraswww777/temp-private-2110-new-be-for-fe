## Краткое описание

Создать макеты второго варианта работы с добавлением задания в пакеты: добавление одного задания в несколько пакетов одновременно с табличным отображением списка пакетов.

---

## Исходное описание задачи

> Данный раздел содержит оригинальное описание задачи для оперативного обращения.

**Источник:** [Замечания от 06.02.2026](https://github.com/taraswww777/temp-private-2110-new-be-for-fe/blob/feature/Masha/docs/Правки%20от%2006.02.md#6406)

### Список заданий. Добавление задания в пакеты

На макетах отрисовать второй вариант работы.

Добавляем одно задание в несколько пакетов, в которых нет добавляемого задания.

При добавлении задания в пакет показывать "Список пакетов" в табличном виде. (по аналогии со "Списком заданий", в форме "Удаление задания из пакетов").

Добавить сортировку по размеру (чтобы пользователь отсортировал по самому легкому пакету и добавил туда задачу)

На демо в понедельник показать оба варианта (на макетах) (текущая реализация и новый формат для сравнения).

Задание можно добавить в 1 или несколько пакетов сразу (массовый выбор)

---

## Цели

- Создать макеты второго варианта добавления задания в пакеты
- Реализовать табличное отображение списка пакетов
- Добавить возможность массового выбора пакетов
- Обеспечить сортировку по размеру

---

## Детальное описание

### Контекст

Текущая реализация добавления задания в пакеты требует улучшения. Необходимо создать второй вариант с более удобным интерфейсом для массового добавления.

### Требования

#### 1. Макеты UI

**Место работы:** [Pixso - Дизайн макеты](https://pixso.t1-pixso.ru/app/design/HjZRf2pFAQMSWhX37B7uAA)

**Требования к макетам:**
- Второй вариант работы с добавлением задания в пакеты
- Табличное отображение списка пакетов (по аналогии со "Списком заданий")
- Возможность массового выбора пакетов (чекбоксы)
- Сортировка по размеру пакета
- Отображение только пакетов, в которых нет добавляемого задания

**Структура таблицы:**
- Колонка выбора (чекбокс)
- Наименование пакета
- Размер пакета
- Статус пакета
- Дата создания
- Другие релевантные поля

#### 2. Функциональность

**Основные возможности:**
- Выбор одного или нескольких пакетов
- Сортировка по размеру (для выбора самого лёгкого пакета)
- Фильтрация пакетов (только те, где нет добавляемого задания)
- Подтверждение добавления

#### 3. Демонстрация

**Требование:**
На демо в понедельник показать оба варианта:
- Текущая реализация
- Новый формат (на макетах)
