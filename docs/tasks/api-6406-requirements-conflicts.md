# Конфликты требований по API формы 6406

Документ фиксирует конфликты требований, выявленные при разборе замечаний (TASK-014) и сопоставлении с уже выполненными задачами (TASK-010, TASK-011, TASK-012, TASK-013).

---

## 1. createdBy: null для «автоматических» заданий

| Аспект | Описание |
|--------|----------|
| **Конфликт** | TASK-011 зафиксировал: при ответе поле `createdBy` всегда заполнено (не null). В TASK-014 поставлен вопрос: «почему null? задание может прийти автоматически». Если продукт захочет явно различать «создано пользователем» (строка) и «создано системой» (null или отдельный флаг), это противоречит решению «всегда не null». |
| **Рекомендация** | Либо оставить текущее решение (пустая строка / системное значение при автоматическом создании), либо явно принять nullable + описание в API и согласовать с TASK-011. |

---

## 2. currency в создании задания

| Аспект | Описание |
|--------|----------|
| **Конфликт** | TASK-014: currency не должно быть обязательным при создании. В коде `createTaskSchema` поле `currency` было обязательным. TASK-012 добавляет currency в ответ (TaskDetailsDto), но не меняет контракт создания. |
| **Рекомендация** | Конфликта нет: сделать currency опциональным только в CreateTaskDto (request); в ответе оставить по TASK-012. *(В TASK-014 реализовано: currency опционально при создании, по умолчанию RUB.)* |
