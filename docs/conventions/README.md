# Соглашения и стандарты проекта

Эта папка содержит документы с соглашениями, стандартами и best practices для разработки проекта.

## 📋 Список документов

### [File Size Conventions](./file-size-conventions.md)
**Дата:** 2026-01-29  
**Статус:** Active  
**Связано с:** [TASK-008](../tasks/TASK-008-standardize-size-storage-and-api.md)

Регламентирует работу с размерами файлов, пакетов и объёмами данных:
- Хранение размеров в БД (bigint, байты)
- Передача размеров в API (number, байты)
- Форматирование для пользователя
- Соглашения по именованию полей
- Утилиты и константы

**Для кого:**
- Backend разработчики (схемы БД, API)
- Frontend разработчики (форматирование, вычисления)
- Все разработчики (общие принципы)

## 🔍 Как использовать

1. **Перед началом работы** — ознакомьтесь с соответствующим документом
2. **При создании нового функционала** — следуйте установленным соглашениям
3. **При code review** — проверяйте соответствие стандартам
4. **При обнаружении проблем** — создайте issue или предложите улучшение

---

## ✍️ Внесение изменений

Чтобы предложить изменения в соглашения:

1. Создайте задачу (TASK) с описанием проблемы
2. Предложите решение с обоснованием
3. Обсудите с командой
4. Обновите документ после утверждения
5. Обновите связанные задачи и код
