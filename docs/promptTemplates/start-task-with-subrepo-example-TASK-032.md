# Пример заполненного промпта: TASK-032 + VTB-533

Это пример шаблона `start-task-with-subrepo.md` с подставленными значениями для задачи TASK-032 и ветки VTB-533 в подрепо. Можно скопировать и отправить ассистенту или использовать короткую форму (см. .cursorrules).

---

Начать работу над задачей **TASK-032** в новой ветке.

**Контекст:**
- Задача в основной репе: **TASK-032** (файл задания: `docs/tasks/TASK-032-status-model-documentation.md`).
- В подрепозитории `temp-private-2110/`:
  - Ветка: `feature/VTB-533/FE-Dokumentaciya-dlya-mokov`
  - Префикс коммитов: **VTB-533**

**Действия:**
1. Создать ветку для задачи в основной репе (если ещё не создана), например `feature/TASK-032-status-model-documentation`.
2. В подрепо `temp-private-2110/` переключиться на ветку `feature/VTB-533/FE-Dokumentaciya-dlya-mokov` (или создать её при необходимости).
3. Открыть описание задачи в `docs/tasks/` и выполнять требования по шагам.
4. Следовать протоколу управления заданиями (`docs/cursorRules/task-management-protocol.md`).

---

⚠️ **КРИТИЧНО: коммиты и пуш в оба репозитория**

После каждой логически завершённой операции (после каждого изменения кода) необходимо выполнить **коммит и пуш в ОБА репозитория**:

1. **Сначала в подрепо** `temp-private-2110/` (путь: `temp-private-2110/`):
   - коммиты на **английском** языке;
   - префикс: **VTB-533:** (например, `VTB-533: Add docs for mocks`).

2. **Затем в основной репо** (корневой каталог проекта):
   - коммиты на **русском** языке;
   - префикс: **TASK-032:** (например, `TASK-032: Обновление подмодуля после документации моков`).

Невыполнение этого требования может привести к потере изменений и проблемам с синхронизацией между репозиториями.

---

Выполнить задачу по шагам из описания TASK-032, коммитить и пушить после каждого логически завершённого изменения в оба репозитория.
